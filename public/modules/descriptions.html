Descriptions

Developing a semantically correct approach to handling descriptions for images and charts, showing clearly the relationship between the two.

When to use

This approach should be used to when illustrations, diagrams, photos, code listings, etc would benefit from a more detailed descriptor to explain. This also includes reference to sources or credits where applicable.

It isn't always neccessary and images with correctly structured alt attributes may also be appropriate.

Figure

The object is a container for a user interface element such as a table, chart, or image and a caption which labels the element

Caption

The object contains descriptive information, usually textual, about another user interface element such as a table, chart, or image.

Alt attributes

Alt attributes should be considered as image replacements, though not solely for the purpose of screenreaders, though their benefit for this use is very high, they can also be used in case a person is on a low bandwidth.

When considering what to use as a description in an alt tag, always think of the context, where is it on the page and what is the image conveying? You can use this simple design tree to help decide whether to use, and WebAIM also has some examples of how best to describe images in alt attributes.

You can add a link from the image to a description further down the page, which can be read by a screenreader, but this could affect the flow for the user. In this instance it is best to use an alt label of just the question. It will enable the screenreader user to understand what is being explained below without adding confusion.
